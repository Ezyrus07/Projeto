1 Executar SQL no Supabase

O que faz: Corrige as políticas de segurança (RLS) que estão impedindo o site de acessar os dados dos usuários.

  1. Acesse: https://app.supabase.com
  2. Entre no seu projeto
  3. Clique em "SQL Editor" no menu lateral
  4. Copie e cole o SQL abaixo:
-- Correção RLS - Tabela Usuários
alter table if exists public.usuários enable row level security;

-- Remove policies antigas
drop policy if exists "usuários_upsert_own" on public.usuários;
drop policy if exists "usuários_update_own" on public.usuários;
drop policy if exists "usuários_select_own" on public.usuários;
drop policy if exists "usuários_select_authenticated" on public.usuários;

-- SELECT (permite usuários autenticados lerem perfis)
create policy "usuários_select_authenticated"
  on public.usuários for select to authenticated using (true);

-- INSERT (permite criar próprio perfil)
create policy "usuários_upsert_own"
  on public.usuários for insert to authenticated
  with check (uid::text = auth.uid()::text);

-- UPDATE (permite atualizar próprio perfil)
create policy "usuários_update_own"
  on public.usuários for update to authenticated
  using (uid::text = auth.uid()::text)
  with check (uid::text = auth.uid()::text);
✅
Como saber que funcionou:
Deve aparecer "Success. No rows returned" ou similar

3 Limpar Cache do Navegador

O que faz: Remove dados antigos e corrompidos que estão fazendo o site falhar.

Escolha uma opção:

Opção A: Limpar manualmente (Chrome/Edge)
  1. Pressione Ctrl + Shift + Delete
  2. Selecione "Todo o período"
  3. Marque: "Cookies" e "Cache"
  4. Clique em "Limpar dados"
